# 贪心算法
- 概念：在每一次都要求局部最优，而不要求全局最优
- 贪心其实不要求知其所以然
  - 但是如何验证？对数器的方式来进行验证，然后通过随机产生器
- **注意**
  - 不要纠结严格证明，浪费时间且收益很低
  - 解法只包含贪心思路的题目，代码量不大
  - 重点是贪心方式的特征也就是题目的特征，而不是证明，累积经验很重要
  - 用对数器来验证正确性就好
  - 如果数据量能够支持非贪心策略，则优先去贪心
  - 广义的贪心无处不在，可能和别的思路结合，可以通过自然智慧想明白，不用纠结证明

## 题目一：最大数
- 由经典问题改动：将一个数组中所有的字符片段都用上，拼成一个字典序最小的字符串
- 有a和b两个字符片段，有两种排序策略：
  - 若a>b，则a在b的后面；反之，a在b的前面
  - 若a.b（代表a拼接上b）> b.a，则采用b.a，反之，采用a.b
  - 上述两种策略中，前一种策略是错误的，后一种策略是正确的
- 证明第二种策略是正确的
  - 首先，该策略具有传递性，即a.b<b.a, b.c<c.b，则有a.c<c.a
    - 定义f(k) = 对应进制^k的位数，比如k = "abc"，则f(k) = 26^3，因为字母为26进制，且abc长度为3
    - a拼接上b = a * f(b) + b
    - 上述的两个条件变成
      - a * f(b) + b <= b * f(a) + a; b * f(c) + c <= c * f(b) + b
      - 经过变换可以得到最终的式子f(c) * a + c <= f(a) * c + a
  - 按照上述策略连接得到的字符串，经过任何一次交换都一定变得更大(传递性)，因为就必定是最小的字符串
- **提醒**：真正的贪心证明实在是很繁琐，所以最好掌握对数器验证

## 题目七：砍竹子II
- 在刚拿到题目的时候可能会认为需要用动态规划来做，但是不行，原因是乘积得到的中间结果必然很大，所以势必要取模
- 但是取模会**丧失数字之间的大小关系**，所以无法判断最大乘积是谁
- 纯观察，我们可以在以下的式子中发现某些规律

n = 4  --> 2 * 2         |  n = 5  --> 3 * 2         |  n = 6  --> 3 * 3
n = 7  --> 3 * 2 * 2     |  n = 8  --> 3 * 3 * 2     |  n = 9  --> 3 * 3 * 3
n = 10 --> 3 * 3 * 2 * 2 |  n = 11 --> 3 * 3 * 3 * 2 |  n = 12 --> 3 * 3 * 3 * 3
**规律**：n % 3 == 0，则为$3 ^ ((n - 4) / 3) * 4$，n % 3 == 1，则为$3 ^((n-2)/3)*2$，n % 3 == 2，则为$3^(n/3)$

- 为加速运算，此处用快速幂来计算power3

## 题目十二：计算数组的任意两个数的差值的绝对值
- 裴蜀定理：最大公约数可以表示成两个数的线性组合
  - 形式化地说，任意整数a，b，存在整数x，y使得ax + by = gcd(a,b)存在
  - 本质上，求b-a或者a-b，然后反复用差值减其中的某个数，就是在不断构造xa + yb的过程，也就是线性组合的过程
  - 则生成的数都应该是gcd(a,b)的倍数
- 所以我们滚动计算所有数组的元素的最大公约数，小于最大值的所有最大公约数的倍数都会最终被加入到数组中
- 要处理两个边界问题
  - 第一是零的讨论，因为在gcd的计算中我们跳过了0。如果数组中曾经有0，那么就直接把0的数量给加上就好；如果没有出现0，那么只要出现过重复数组，就要再+1
  - 第二是重复数字的讨论，因为gcd一定会把原来数组中的元素生成时都包含一遍，所以要统计不止出现一次的数字，将数字的出现次数-1，再加回答案上

## 题目十五：买门票的最大价格
- 传统视角下，我们会以整体人数为角度来衡量门票钱，比如
  - 1个人来玩，那么1个人交 10 - 2*1 = 8
  - 2个人来玩，那么2个人交 2 * (10 - 2 * 2) = 12
  - 3个人来玩，那么3个人交 3 * (10 - 2 * 3) = 12
  - 4个人，交 4 * 2 = 8
  - 5个人，交0元
  - 但这种决策和衡量的角度并不利于在线决策，换句话说不利于贪心，因为每次状态都和整个人数有关，牵一发而懂全省
- 所以我们更换视角，以单个人数带来的利润增量为衡量指标，比如
  - 1个人来，这个人交8元
  - 2个人来，新来的人交4元，总共12元
  - 3个人来，新来的人交0元，总共12元
  - 4个人来，新来的人交-4元，总共8元
  - 5个人来，新来的人交-8元，总共0元
  - 这个时候的思维惯性可能会去到差分数组，但是差分数组的坏处在于，有m个项目，每个项目都要去跑n遍，m*n的复杂度无法接受
- 那么则借助大根堆，和公式来计算当前人到达的时候需要交多少钱
  - 设当前人数为x，第x+1到来的时候，需要额外支付的费用为 $Bi - (x+1)*Ki - x*Ki$

## 题目十六：k份均分的数组平均值最小值
- 我们想要的结果是，让每个集合的平均值都尽可能小
- 贪心策略是，只要在某个集合内，有一个小数和一个大数，大数就一定会拉高小数的平均值
  - 所以我们让前k-1个集合都只有一个数，然后让剩下的数都在最后一个集合中，这样就只有最后一个大数

## 题目十七：执行所有任务的初始电量
- 非常巧妙的一道题，集成了很多算法设计的思想，需要好好想想
- 我们最终的目的是，找到一个最小的初始电量，从这个电量开始往下减，最后的电量剩得越少越好，最优解当然是0
- 所以我们想当然地可以从0开始逆推答案，这样会更加方便
- 我们有两个衡量因素，一个是耗费，一个是至少，一一分析
  - 对于耗费来说，控制至少是一样的。考虑这么两个例子，A: [8,10]，B: [6,10]
    - 当从耗费多的开始倒推，0为了经过A，至少要有10的电量；10要经过B，至少要有16的电量，16是满足10的阈值要求的
    - 当从耗费少的开始倒推，0为了经过B，至少要有10的电量；10要经过A，至少要有18的电量，18是满足10的阈值的
    - 所以从耗费多的开始倒推是更优解，这也符合我们的直觉，开始加的越多，越有可能越过某些至少高的情况
  - 但是有反例，考虑至少不一样的情况。A: [8,12]，B: [6,8]
    - 当从耗费多的开始倒推，0为了经过A，至少要有12的电量；12要经过B，至少要有18的电量，18满足8的阈值
    - 当从耗费少的开始倒推，0为了经过B，至少要有8的电量；8为了经过A，至少要有16的电量，16满足12的阈值
    - 诶？怎么从耗费多的开始推不行了？
  - 往下，我们考虑至少。控制耗费一样。考虑这么两个例子，A: [5, 10]，B: [5,20]
    - 当从至少更少的开始倒推，0为了经过A，至少需要10电量，为了经过B，至少需要max(10+5,20)=20电量
    - 当从至少更多的开始倒推，0为了经过B，至少需要20电量，为了经过A，至少需要25
    - 所以从至少更少的开始倒推是最优解，这同样符合我们的直觉，门槛从高到低，取门槛更高的便于通过
  - 但是有反例，考虑耗费不一样的情况。A: [6,10]，B: [18,20]
    - 当从至少更少的开始倒推，0为了经过A，至少需要10电量，为了经过B，至少需要28
    - 当从至少更多的开始倒推，0为了经过B，至少需要20电量，为了经过A，至少需要26
    - 诶？怎么从至少少的开始倒推不行了？
- 总结上述推导过程，可以发现，我们想要至少尽可能少，耗费尽可能多。
- 所以，我们的倒推依据就应该是，耗费-至少，越大则在倒推位置中排靠前

## 题目十九：数组的最小偏移量
- 先来考虑规律，对于奇数来说，只能够先*2，然后变成偶数，就只能再÷2，换言之奇数只能震荡
- 对于偶数来说，如果÷2之后还是偶数，就还可以继续除，如果变成奇数了，就只能震荡了，换言之就到底了
- 所以我们的策略是
  - 先给所有奇数一个向上提升的机会
  - 然后从最大的偶数开始，往下降，每降一次就计算一次差值，直到降为奇数
- 考虑用什么数据结构，因为我们需要同时获得最大值和最小值，所以有序表set是一个不错的选择

## 题目二十一：使数组相似的最少操作次数
- 考虑规律，这些操作nums[i] = nums[i] + 2，nums[j] = nums[j] - 2，规律是不会改变奇偶性，也就是说，如果nums一定能够变成target，则num和target的奇偶数一定一样
- 所以我们可以分组，将奇数一组，偶数一组，然后从小到大排序，因为贪心思想，对应的下标一定是对应的要变动的数
- 怎么计算要动几次？忽略正负性的改变，直接看绝对值，而一次操作会带来4的改变，所以用这个绝对值的累加去除上4就可以


## 题目二十二：知识竞赛
- 题干信息可以进行两次简化，第一次是计算的(A_i + A_j) / 2，其实/2或者不/2本身没有区别，所以为了节约成本就不除2
- 第二次是，我们可以发现，如果用暴力解法，其实不同人的排序关系有任何影响，只需要到达某个位置时将左边的所有人都加在一起取最大就好了
  - 但是这样的复杂度是O(n^2)，我们不允许这种复杂度出现
  - 应该怎么给当前这个序列进行一个排序，从而能让我们在一个可接受的（比如O(logn)）的时间内找到当前位置的最高瓶颈？

    - 这样排序有什么道理呢，让我们想想，假设有这么一个人，推理能力和阅读能力是[20, 15]，阅读能力一定是她的瓶颈
    - 如果她的左边有一个人，推理能力大于阅读能力，则这个结论不证自明
    - 如果她的左边有一个人，阅读能力大于推理能力，按照上述排序规则排序之后，阅读能力带来的差值一定<=5，说明阅读能力还会是瓶颈

## 题目二十四：最小加油次数
