# 并查集
## 使用场景
- 一开始每个元素都拥有自己的集合，在自己的集合里只有这个元素自己
- 时间复杂度：每个操作单次调用的均摊时间复杂度为O(1) —— 反阿克曼函数
## 类方法
- int find(a)  查找某个元素属于的类别
- bool isSameSet(a, b)  检查某两个元素是否属于同一个类别
- void Union(a, b)  合并a b两个元素（小挂大）
## 实现结构
- parent数组: 存储某个位置元素的家长节点
- size数组: 记录某个节点（作为根）时的树的大小 —— 可作为合并依据
## 优化手段
- 小挂大：依靠size数组来合并 （可以省略：和秩的概念相关） 
- 扁平化：在find过程中直接进行路径压缩

  - 和例1是同一类题目，需要发现合并石头的次数其实就是总石头数 - 最后剩余的总集合数

- 例5.知晓秘密的专家
  - 关键点在于学会打标签和处理标签，标签不是要所有的元素都打上，只需要将每个集合的根节点打上即可；注意在搜寻过程中的flag确认都应该是find函数返回的index，而不是element本身
- 例6.好路径数量
  - 这里用到了两个标签数组，问题在于，怎样通过处理边来完成好路径的统计？用到两个很妙的方法
  - 第一个是，合并时统计max值，并记录拥有max值的个数，因为好路径能否纳入答案其实并不关注除了最大值之外的其他路径，只要确保起点终点答案一样，且中途是更小的就行
  - 第二个是，处理边的顺序应该是离线决定好的，即按照两侧点权值最大值，从小到大依次处理，这样不会跳解或者错解
